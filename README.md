# Bubbs-Talks — Capacitor iOS shell

Capacitor wrap around the HTML AAC prototype. Used because React Native +
Hermes does not run on iPhone 17 Pro Max / iOS 26 (PAC pointer
authentication crash). WKWebView uses Apple's own JavaScriptCore which has
no PAC issues.

## Layout

- `www/index.html` — the AAC interface (vocabulary tiles, sentence flow,
  audio output, caregiver mode). This is the entire app logic.
- `capacitor.config.json` — bundle ID, app name, web directory.
- `codemagic.yaml` — Codemagic CI workflow that builds iOS app on macOS,
  signs with Apple credentials, submits to TestFlight.
- `ios/` — generated by Codemagic during `npx cap add ios`. Don't commit.

## Build flow

1. Push to `main` → Codemagic auto-builds via `codemagic.yaml`.
2. Codemagic runs on macOS, executes `npx cap add ios && npx cap sync`,
   then `xcode-project build-ipa`.
3. Built IPA auto-submitted to TestFlight.
4. Once tested, manually promote to App Review via App Store Connect.
